# Development Notes

## Текущее состояние проекта

- Chrome-расширение подключается к открытой вкладке Яндекс Музыки.
- Локальный сервер связывает расширение и телефон через QR pairing.
- Телефонный пульт умеет:
  - play/pause;
  - предыдущий и следующий трек;
  - лайк и дизлайк;
  - перемотку;
  - регулировку громкости;
  - показ следующих треков;
  - выбор вариантов `Моей волны`.

## Важные технические решения

- Для работы используется адаптированная копия `YmExternalAPI` в `extension/externalAPI`.
- API неофициальное и работает через внутренние объекты открытой страницы Яндекс Музыки.
- `replace-webpack.js` перехватывает загрузку внутренних модулей страницы и получает `controller` и `playbackController`.
- Варианты `Моей волны` берутся динамически из `rootStore.wheel.items`.
- Выбор варианта `Моей волны` реализован через:
  - `Controller.contextController.factory.createContext(...)`
  - `Controller.playContext(...)`
- Для `diversity:reshuffle` создается новый seed вида:
  - `diversity:reshuffle_<timestamp>`
- Кнопка `next` защищена:
  - если следующий трек еще не подгружен Яндекс Музыкой, команда не отправляется;
  - это сделано, чтобы не ломать внутреннюю очередь `Моей волны`.
- Громкость на телефоне меняется только при перетаскивании бегунка, чтобы случайный тап по полоске не менял громкость.
- CLI сервера включается только через:
  ```powershell
  $env:ENABLE_CLI="1"
  npm start
  ```
- На Ubuntu сервер проверен через `systemd`; в простое нагрузка близка к нулю.

## Известные особенности

- Варианты `Моей волны` появляются только после того, как сама страница Яндекс Музыки загрузила `wheel.items`.
- Если на телефоне написано `Варианты пока не загружены`, нужно убедиться, что на ПК открыт экран с вариантами `Моей волны`.
- В текущей реализации `prev` для `Моей волны` идет только внутри укороченного окна последних треков, а на сайте Яндекс Музыки можно идти назад по более длинной истории.
- Улучшение `prev` для `Моей волны` - следующий логичный кандидат на доработку.

## Возможные будущие задачи

- Сделать `prev` для `Моей волны` ближе к поведению сайта Яндекс Музыки.
- Исследовать добавление текущего трека в пользовательские плейлисты.
- При необходимости добавить отдельную историю недавно проигранных треков.
- При изменениях на стороне Яндекс Музыки обновлять слой совместимости в `extension/externalAPI`.

## Как продолжать работу в новом чате

1. Сначала прочитать:
   - `README.md`
   - `DEVELOPMENT.md`
2. Затем проверить актуальное состояние репозитория.
3. Основной следующий вопрос:
   - как безопасно переделать `prev` для `Моей волны`, чтобы использовать реальную историю Яндекс Музыки, а не только последние треки.

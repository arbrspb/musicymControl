# MusicYM Control

MusicYM Control — неофициальный локальный пульт для веб-версии Яндекс Музыки.

Проект состоит из двух частей:

- `extension` — Chrome-расширение, которое подключается к открытой вкладке Яндекс Музыки и передаёт команды плееру;
- `server` / `server_ubuntu` — локальный HTTP/WebSocket-сервер, который связывает расширение и телефон через QR pairing.

## Отказ от ответственности

Проект не является официальным продуктом Яндекса, не связан с Яндексом и не одобрен Яндексом.

Для работы используется адаптированная копия API из [YmExternalAPI](https://github.com/Night-Soft/YmExternalAPI) автора Night-Soft. Код находится в папке `extension/externalAPI` и опирается на внутренние объекты страницы Яндекс Музыки. Поэтому отдельные возможности могут перестать работать после изменений на стороне сервиса.

Проект работает локально и не собирает cookies, токены или данные аккаунта пользователя.

## Возможности

- play / pause;
- предыдущий и следующий трек;
- лайк и дизлайк текущего трека;
- перемотка текущего трека;
- управление громкостью с телефона;
- список следующих треков с возможностью выбрать трек;
- выбор вариантов `Моей волны`;
- запуск `Моей волны по треку` для текущей композиции;
- сброс к обычной `Моей волне`;
- веб-пульт на телефоне по QR-коду;
- автоматическое переподключение телефона после возврата во вкладку, восстановления сети или повторного открытия страницы;
- подробные диагностические логи для разборов редких сбоев;
- автоматический выбор локального IP-адреса ПК с обходом VPN/виртуальных интерфейсов, насколько это возможно.

## Как это устроено

```text
Телефон  <->  локальный сервер  <->  Chrome-расширение  <->  вкладка Яндекс Музыки
```

Телефон не управляет Яндекс Музыкой напрямую. Он отправляет команду серверу, сервер передаёт её расширению, а расширение вызывает внутреннее API открытой страницы Яндекс Музыки.

## Требования

- Node.js 20 или новее;
- Google Chrome 116 или новее;
- телефон и ПК в одной локальной сети;
- входящий доступ к порту сервера, по умолчанию `8099`.

## Быстрый запуск

Из папки сервера:

```powershell
cd "Ваш путь\server"
npm install
npm start
```

После запуска сервер покажет адреса:

```text
[server] local:  http://127.0.0.1:8099
[server] public: http://192.168.x.x:8099
[server] ws:     ws://192.168.x.x:8099/ws
```

- `local` нужен расширению на том же ПК;
- `public` нужен телефону в локальной сети.

## Установка расширения в Chrome

1. Откройте `chrome://extensions/`.
2. Включите режим разработчика.
3. Нажмите `Загрузить распакованное расширение`.
4. Выберите папку `extension`.
5. Откройте `https://music.yandex.ru/`.
6. Нажмите на иконку расширения.
7. Проверьте адрес сервера, обычно это `http://127.0.0.1:8099`.
8. Создайте QR pairing.
9. Отсканируйте QR-код телефоном.

Если расширение перезагружалось или обновлялось, обновите вкладку Яндекс Музыки.

## Как пользоваться

1. Запустите сервер.
2. Откройте Яндекс Музыку в Chrome.
3. Откройте popup расширения и создайте QR pairing.
4. Откройте пульт на телефоне по QR-коду.
5. Управляйте воспроизведением с телефона.

### Моя волна по треку

Кнопка `Моя волна по треку` запускает рекомендации от текущего трека через внутренний API Яндекс Музыки, а не через клик по DOM-элементам страницы. Это делает механизм менее зависимым от визуальной верстки сайта.

Функция работает как для трека из коллекции, так и для трека, который уже играет внутри `Моей волны`.

### Сбросить волну

Кнопка `Сбросить волну` возвращает обычную `Мою волну` с seed `user:onyourwave`.

Нативный крестик в интерфейсе Яндекс Музыки дополнительно передаёт `queue` уже сыгранных треков. В текущей реализации это сознательно не повторяется: для самого сброса `queue` не обязателен, а без необходимости усложнять логику проекта не стоит. Если позже выяснится, что после сброса рекомендации заметно хуже продолжаются или чаще повторяются треки, поддержку `queue` можно добавить отдельной задачей.

## Если сервер запущен на другом устройстве

По умолчанию расширение уже разрешает подключение к `127.0.0.1` и `localhost`, то есть к серверу на том же ПК, где открыт Chrome.

Если сервер работает на другом устройстве в локальной сети, например на Ubuntu-сервере, добавьте его адрес в `extension/manifest.json` в `host_permissions`.

Пример для сервера `192.168.1.120`:

```json
"http://192.168.1.120/*"
```

После изменения:

1. Перезагрузите расширение в `chrome://extensions/`.
2. Обновите вкладку Яндекс Музыки.
3. В popup расширения укажите адрес этого сервера, например:

```text
http://192.168.1.120:8099
```

## Запуск сервера на Ubuntu

Можно использовать папку `server_ubuntu`.

Пример размещения:

```bash
/opt/musicym-control/server
```

Первый запуск:

```bash
cd /opt/musicym-control/server
npm install
npm start
```

Проверка:

```bash
curl http://127.0.0.1:8099/api/health
```

Ожидаемый ответ:

```json
{
  "ok": true
}
```

### Автозапуск через systemd

Пример unit-файла:

```ini
[Unit]
Description=MusicYM Control Server
After=network.target

[Service]
Type=simple
User=musicym
WorkingDirectory=/opt/musicym-control/server
ExecStart=/usr/bin/node /opt/musicym-control/server/ws_server.js
Restart=always
RestartSec=3
Environment=PUBLIC_HTTP_ORIGIN=http://192.168.1.120:8099

[Install]
WantedBy=multi-user.target
```

После создания службы:

```bash
sudo systemctl daemon-reload
sudo systemctl enable musicym-control
sudo systemctl restart musicym-control
sudo systemctl status musicym-control
```

## Debug и логирование

У проекта есть два уровня логов.

### 1. Обычный серверный лог

`server.log` нужен для общей диагностики: подключений, HTTP-запросов, команд и ошибок.

Включение при ручном запуске:

```powershell
$env:DEBUG="1"
npm start
```

Если нужно также выводить debug в консоль:

```powershell
$env:DEBUG="1"
$env:DEBUG_CONSOLE="1"
npm start
```

### 2. Trace-лог событий

Файлы вида `trace-YYYY-MM-DD.jsonl` нужны для точного разбора цепочки событий между телефоном, сервером, расширением и плеером.

Они помогают расследовать редкий глюк с возможным двойным воспроизведением: не только увидеть, что в какой-то момент появились два аудиопотока, но и восстановить, что произошло перед этим — какую команду отправил телефон, что расширение видело до команды, что стало сразу после неё и что изменилось через короткую задержку.

В trace попадают, среди прочего:

- нажатия с телефона;
- полученные команды;
- выбранная вкладка Яндекс Музыки;
- состояние до и после команды;
- отложенные снимки состояния после `80` и `300` мс;
- события плеера;
- изменение количества реально играющих `<audio>`-элементов.

Полезные события при поиске редкого сбоя:

```text
phone.command.click
command.received
extension.command.received
extension.player.tabs.before-command
extension.player.tab.selected
player.command.before
player.command.after
player.command.after-delay
audio.playing-count.changed
audio.multiple-playing
EVENT_STATE
EVENT_PROGRESS
```

Важно: `audioPlayingCount` — полезный сигнал, но не единственный источник истины. Если он остался `0`, это ещё не доказывает отсутствие звука. Для причины сбоя важнее вся последовательность событий вокруг команды.

## Частые проблемы

### Телефон пишет «ожидание pairing»

В `server.log` после открытия пульта с телефона должны появиться строки:

```text
New WebSocket connection established
handleHello from phone
Socket bound: role=phone
```

Если есть только:

```text
HTTP request: GET /remote
```

значит страница открылась, но WebSocket не подключился.

Проверьте:

- телефон и ПК находятся в одной сети;
- firewall не блокирует порт `8099`;
- QR создан после последнего запуска сервера;
- адрес в QR начинается с локального IP ПК, например `http://192.168.x.x:8099`.

### После возврата в браузер телефон показывает переподключение

Пульт умеет восстанавливаться сам после `visibilitychange`, `pageshow` и `online`. Если связь всё же зависла, сначала подождите несколько секунд. Если не помогло — обновите страницу пульта.

### Расширение подключилось, а телефон нет

В логе будет:

```text
handleHello from extension
Socket bound: role=extension
```

но не будет `handleHello from phone`.

Проверьте сеть телефона, firewall и адрес в QR-коде.

### Телефон подключился, но трек не отображается

Проверьте:

- открыта ли вкладка `music.yandex.ru`;
- обновлялась ли вкладка после перезагрузки расширения;
- нет ли ошибок content script в DevTools страницы Яндекс Музыки.

### Кнопка «вперёд» временно не нажимается

Это нормальная защита. Иногда Яндекс Музыка ещё не успевает подготовить следующий трек. Пока следующий трек не готов, команда `next` не отправляется, чтобы не сломать внутреннюю очередь плеера.

### После обновления расширения появились ошибки `Extension context invalidated`

Обычно это не критично. Chrome перезагрузил расширение, а старая вкладка Яндекс Музыки ещё держит прежний content script. Решение — обновить вкладку Яндекс Музыки.
